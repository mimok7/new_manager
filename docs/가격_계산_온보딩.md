가격 계산 온보딩 가이드

목적: 다른 개발자가 로컬에서 가격 계산 중앙화 작업을 이어서 할 때 참고할 체크리스트와 명령어를 제공합니다. 기존 `docs/가격_계산_지침.md`를 먼저 읽으세요.

Prerequisites
- Git (최신)
- Node.js >= 20.11
- pnpm (v9 추천)
- Windows PowerShell (또는 bash)

1) 저장소 복제 및 의존성 설치

```powershell
# 리포지토리 복제
git clone https://github.com/mimok7/shtplatform.git
cd sht-platform

# 의존성 설치
pnpm install
```

2) 개발 서버 실행 (예: manager 앱)

```powershell
pnpm --filter @sht/manager dev
# 또는 quote 앱
pnpm --filter @sht/quote dev
```

3) 코드 위치 및 지침 확인
- 중앙 계산 로직: `packages/domain/src/pricing`
- 정책 문서: `docs/가격_계산_지침.md`
- 중복 파일 위치(예시):
  - `apps/*/lib/cruisePriceCalculator.ts`
  - `apps/*/src/lib/cruisePriceCalculator.ts`

4) 자동 교체 스크립트 사용 (윈도우 PowerShell)
- 아래 스크립트 `scripts/replace-cruise-with-wrapper.ps1`는 원본 파일을 `.bak`로 백업하고, 공용 모듈로 대체합니다.

```powershell
# 실행 예
cd z:\sht-platform
powershell -ExecutionPolicy Bypass -File scripts\replace-cruise-with-wrapper.ps1
```

스크립트를 직접 실행하기 전에 수정사항을 검토하세요.

5) 수동 확인 (간단한 체크)

```powershell
# 중복된 cruisePriceCalculator 파일 검색
Get-ChildItem -Path . -Include cruisePriceCalculator.ts -Recurse | Select-Object FullName

# 변경된 파일 확인
git status --porcelain
```

6) 테스트 및 타입체크

```powershell
# 타입검사 (모노레포)
pnpm run typecheck

# 개별 패키지 테스트 (예: domain)
pnpm --filter @sht/domain test
```

7) 마이그레이션 후 작업
- 각 앱에서 정상 동작 확인 후, 중복 파일을 삭제하거나 `deprecated` 폴더로 이동하세요.
- PR 템플릿에 `pricing: migrate to @sht/domain/pricing`를 제목에 포함하세요.

문제가 발생하면 `#dev-pricing` 채널에 로그와 `git status` 출력 결과를 공유해 주세요.

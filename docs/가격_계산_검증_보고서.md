## 🔍 상세 검증 결과

### 1️⃣ **호텔 (Hotel) — 🔴 주요 불일치**

#### DB 스키마
```
hotel_price 테이블:
- hotel_price_code (text, NOT NULL) — 가격 코드 PK
- base_price (numeric, NOT NULL) — 1박 기본 요금
- extra_person_price (numeric, NULL) — 추가 인원 요금
- start_date (date, NOT NULL) — 유효 시작일
- end_date (date, NOT NULL) — 유효 종료일
- weekday_type (text, NULL) — 요일별 가격 (ALL, WEEKDAY, WEEKEND)
- occupancy_max (integer, NULL) — 최대 인원
- include_breakfast (boolean, NULL) — 조식 포함 여부
```

#### 기존 계산 (updateQuoteItemPrices.ts)
```typescript
// 호텔 계산: hotel_price_code로 base_price 조회
const { data: priceData } = await supabase
  .from('hotel_price')
  .select('base_price')
  .eq('hotel_price_code', hotelData.hotel_code);

unitPrice = priceData[0].base_price || 0;  // 박수 미적용!
// 결과: base_price만 저장 (총 요금이 아님)
```

**문제:**
- ❌ **박수 미고려**: base_price만 반환 (1박 요금만)
- ❌ **추가 인원 미계산**: extra_person_price 미사용
- ❌ **요일별 가격 미적용**: weekday_type 미사용

#### 신규 계산 (hotel.ts)
```typescript
const nights = getNightCount(checkin_date, checkout_date);
const subtotal = base_price * nights * room_count;  // 박수 × 객실수 고려

return {
  base_price,     // 1박 요금
  nights,         // 박수
  room_count,     // 객실 수
  subtotal,       // base_price × nights × room_count
  price_breakdown: { ... }
};
```

**개선:**
- ✅ **박수 계산 추가**: checkout - checkin 자동 계산
- ✅ **객실수 곱셈**: room_count 반영
- ✅ 가독성 개선: 세부 breakdown 제공

#### 📌 **결론: 기존은 불완전, 신규가 올바름**
```
기존: unitPrice = 100 (1박 요금만) → 4박 2실 시 총액 100원? ❌
신규: subtotal = 100 × 4nights × 2rooms = 800 ✅
```

**권장:** 신규 계산기 적용, 기존 데이터 재계산 필요

### 2️⃣ **공항 (Airport) — 🟢 호환 (재검증 필요)**

#### DB 스키마
```
airport_price 테이블:
- id (uuid, NOT NULL) — PK
- airport_code (text, NOT NULL) — 공항 코드
- service_type (text, NOT NULL) — 서비스 유형 (pickup, dropoff)
- vehicle_type (text, NOT NULL) — 차량 타입 (SUV, VAN, etc)
- route (text, NOT NULL) — 루트 설명
- price (integer, NOT NULL) — 가격 (정수)
- year (integer, NULL) — 연도
- is_active (boolean, NULL) — 활성 여부
```

#### 기존 계산 (updateQuoteItemPrices.ts)
```typescript
// 공항 계산: airport_code → price × passenger_count
const { data: priceData } = await supabase
  .from('airport_price')
  .select('price')
  .eq('airport_code', airportData.airport_code);

unitPrice = (priceData[0].price || 0) * (airportData.passenger_count || 1);
// 결과: price × passenger_count
```

#### 신규 계산 (airport.ts)
```typescript
const quantity = Math.max(1, input.quantity ?? 1);  // 기본값 1

const { data, error } = await query
  .from('airport_price')
  .select('price')
  .eq('airport_code', input.airport_code)
  .limit(1)
  .maybeSingle();

const unit_price = Number(data?.price ?? 0);
const subtotal = unit_price * quantity;  // price × quantity
// 결과: unit_price × quantity
```

#### 📌 **호환성 분석**
```
변수 매핑:
- 기존 passenger_count = 신규 quantity ✅
- 기존 price = 신규 unit_price ✅
- 공식: (price × count) = (unit_price × quantity) ✅
```

**호환성:** 🟢 **동일한 공식**

**주의사항:**
- ⚠️ **필드명 차이**: `airport_price_code` vs `airport_code`
  - 신규는 두 가지 모두 지원 (유연함)
- ⚠️ **service_type 필터링 미확인**: 기존에서 service_type 필터링 여부 확인 필요

### 3️⃣ **렌트카 (Rentcar) — 🟡 부분 호환 (수정 필요)**

#### DB 스키마
```
rentcar_price 테이블:
- id (uuid, NOT NULL) — PK
- rent_code (text, NULL) — 렌트카 코드
- car_category_code (text, NULL) — 카테고리 코드
- vehicle_type (text, NULL) — 차량 타입
- route (text, NULL) — 루트
- price (integer, NULL) — 1대 가격

- duration_hours (integer, NULL) — 대여 시간
- way_type (text, NULL) — 편도/왕복
```

#### 기존 계산 (updateQuoteItemPrices.ts) — **로직 2가지 분리**
```typescript
// Case 1: 투어 내 차량 (service_type === 'tour_car')
unitPrice = (priceData[0].price || 0) * (carData.car_count || 1);
// 결과: price × car_count (차량 대수 고려)

// Case 2: 독립 렌트카 (service_type === 'rentcar')
unitPrice = priceData[0].price || 0;
// 결과: price만 (차량 대수 미고려!) ❌
```

**문제:**
- ❌ **Case 2 불완전**: 렌트카 전용에서 quantity 미적용
- ⚠️ **로직 이중화**: 투어 차량과 렌트카 로직이 다름

#### 신규 계산 (rentcar.ts)
```typescript
const quantity = Math.max(1, input.quantity ?? 1);

const { data, error } = await this.supabase
  .from('rentcar_price')
  .select('price')
  .eq('rent_code', input.car_price_code)
  .limit(1)
  .maybeSingle();

const unit_price = Number(data?.price ?? 0);
const subtotal = unit_price * quantity;  // price × quantity
// 결과: price × quantity (대수 고려)
```

#### 📌 **호환성 분석**
```
기존 Case 1 (투어 차량): price × car_count ✅
신규: price × quantity ✅
→ car_count = quantity로 매핑 가능

기존 Case 2 (독립 렌트카): price만 사용 ❌
신규: price × quantity (개선)
→ **기존 데이터와 호환 안 됨** (박수/대수 재계산 필요)
```

**권장:** 신규 계산기 적용, 기존 렌트카 예약 데이터 검토


#### 기존 계산 (updateQuoteItemPrices.ts)
```typescript
// 투어 계산: tour_pricing의 price_per_person × participant_count
const { data: priceData } = await supabase
  .from('tour_pricing')
  .select('price_per_person')
  .eq('pricing_id', tourData.tour_code);

unitPrice = (priceData[0].price_per_person || 0) * (tourData.participant_count || 1);
// 결과: price_per_person × count (성인/아동 미구분)
```

#### 신규 계산 (tour.ts) — **성인/아동 분리**
```typescript
const adult_count = Math.max(0, input.adult_count ?? 0);
const child_count = Math.max(0, input.child_count ?? 0);

const { data, error } = await this.supabase
  .from('tour_pricing')
  .select('tour_code, adult_price, child_price');

// 결과: adult_price × adult + child_price × child (성인/아동 구분)
```

#### 📌 **주요 문제: DB 스키마 불일치**

**기존 테이블 구조:**
```sql
SELECT 'price_per_person' FROM tour_pricing;
-- 성인/아동 가격이 별도 컬럼으로 없음
```

**신규 계산기 기대 구조:**
```typescript
// 신규가 찾는 컬럼:
.select('tour_code, adult_price, child_price')

// 실제 테이블에 없음! ❌
```

#### 📌 **호환성 분석**
```
기존: price_per_person (1인당 가격) × count ✅ 구현됨
신규: adult_price × adult + child_price × child ❌ 테이블 구조 미지원

→ **신규 계산기를 기존 구조에 맞게 수정 필요**
```

**필요한 조치:**
1. **Option A: 신규 계산기 수정** (권장)
   ```typescript

   ```

2. **Option B: DB 스키마 추가** (장기)

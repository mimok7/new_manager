## 📝 데이터 입력 방식

### 시즌별 가격 관리

**방식**: 같은 객실을 `schedule_type` 다르게 저장
- `schedule_type = '1N2D-S1'`: 시즌 1 (정가) - 8행
- `schedule_type = '1N2D-S2'`: 시즌 2 (할인) - 8행
- `valid_from / valid_to`: 각 시즌의 정확한 기간 입력

**장점**:
- 시즌별로 명확히 구분
- 향후 시즌 3 추가 용이 (다른 할인율)
- 가격 변경 추적 가능

### 선택 케이스별 가격 적용

| 선택 기준 | 적용 로직 |
|----------|---------|
| valid_year | 2026 고정 |
| 선택 날짜가 시즌 1 범위 내 | schedule_type = '1N2D-S1' 가격 선택 |
| 선택 날짜가 시즌 2 범위 내 | schedule_type = '1N2D-S2' 가격 선택 |
| 휴일 추가 | holiday_date에 일치하면 surcharge_per_person 추가 |
| 토요일 추가 (6-8월) | 자동 판단 또는 선택 옵션으로 표시 |

# 크루즈 데이터 입력 지침 (SQL 자동 생성용)

## 1. 표 형식 (필수)

### 1.1 객실요금 표

**열 순서 (고정)**:
```
객실명 | 성인 | 아동 | 엑스트라 | 싱글차지
```

**행 구성**:
- 각 줄 = 하나의 객실
- 만약 아동 불가 → "불가" 또는 "-" 표시
- 만약 엑스트라 없음 → 빈 칸 또는 "-" 표시

**예시**:
```
| 객실명 | 성인 | 아동 | 엑스트라 | 싱글차지 |
|--------|------|------|--------|--------|
| 프리미어 스위트 (1층) | 5,300,000 | 3,300,000 | 4,700,000 | 8,450,000 |
| 프리미어 스위트 트리플 (1층) | 5,300,000 | 불가 | 불가 | 불가 |
| 프리미어 스위트 (2층) | 5,600,000 | 5,000,000 | - | 8,900,000 |
```

### 1.2 휴일 추가요금 표

**열 순서 (고정)**:
```
기간/날짜 | 추가요금
```

**행 구성**:
- `-` 포함 = **기간** (예: `2026/04/30 - 2026/05/01`)
- `-` 없음 = **단일 날짜** (예: `2026/12/24`)

**예시**:
```
| 기간/날짜 | 추가요금 |
|---------|--------|
| 2026/04/30 - 2026/05/01 | 1,200,000 |
| 2026/12/24 | 1,200,000 |
| 2026/12/31 | 1,200,000 |
```

## 2. 데이터 매핑 규칙

### 2.1 객실요금 → DB 컬럼

| 표 열 | DB 컬럼 | 설명 |
|------|--------|------|
| 객실명 | `room_type` | 한글 객실명 |
| 성인 | `price_adult` | 성인 1인 요금 |
| 아동 | `price_child` | 아동 1인 요금 |
| 엑스트라 | `price_extra_bed` 또는 `price_child_extra_bed` | 엑스트라베드 요금 |
| 싱글차지 | `price_single` | 1인 싱글차지 |

### 2.2 아동 데이터 규칙

**원칙**: 각 객실의 **아동값은 성인값과 별도로 반복 저장**

**DB 저장**:
- 각 객실마다 **성인/아동 2행**이 아니라 **1행**
- `price_adult` = 표의 성인값
- `price_child` = 표의 아동값

**예외**:
- 아동 불가 객실 → `price_child = NULL`
- 특정 객실만 아동 불가입니다.

### 2.3 엑스트라베드 규칙

**구분**:
- 성인 엑스트라베드만 있음 → `price_extra_bed`
- 아동 엑스트라베드까지 있음 → `price_child_extra_bed` (용도별 구분)

**불가 표시**:
- 엑스트라 불가 객실 → `extra_bed_available = false`
- `price_extra_bed = NULL`

### 2.4 싱글차지 규칙

**원칙**: 모든 객실이 싱글차지 보유

**예외**:
- 특정 객실(예: 트리플)이 싱글 불가 → `price_single = NULL`

## 3. 휴일 추가요금 저장 방식

### 3.1 기간형 휴일 (- 포함)

**예**: `2026/04/30 - 2026/05/01`

**저장 방법**: 
- `cruise_holiday_surcharge` 테이블에 **1행 저장** (기간 표시)
  - `holiday_date = 2026-04-30`, `holiday_date_end = 2026-05-01`
  - `surcharge_per_person = 1,200,000`

### 3.2 단일날짜 휴일 (- 없음)

**예**: `2026/12/24`

**저장 방법**:
- `cruise_holiday_surcharge` 테이블에 **1행 저장**
  - `holiday_date = 2026-12-24`, `holiday_date_end = NULL`
  - `surcharge_per_person = 1,200,000`

## 4. SQL 생성 템플릿

### 4.1 기본 구조

```sql
-- Step 0: 기존 데이터 삭제
DELETE FROM cruise_rate_card 
WHERE cruise_name = '[크루즈명]'
  AND valid_year = 2026;

DELETE FROM cruise_holiday_surcharge
WHERE cruise_name = '[크루즈명]'
  AND valid_year = 2026;

-- Step 1: 객실요금 삽입
INSERT INTO cruise_rate_card 
  (cruise_name, schedule_type, room_type, room_type_en, 
   price_adult, price_child, price_infant, price_extra_bed, price_child_extra_bed, price_single, 
   extra_bed_available, valid_year, currency, is_active, valid_from, valid_to, notes)
VALUES 
  -- 객실 1
  ('[크루즈명]', '1N2D', '[객실명]', '[영문명]',
   [성인], [아동], NULL, [엑스트라], NULL, [싱글],
   [true/false], 2026, 'VND', true, '2026-01-01', '2026-12-31', '[비고]'),
  
  -- 객실 N
  ...;

-- Step 2: 휴일 추가요금 삽입
INSERT INTO cruise_holiday_surcharge 
  (cruise_name, schedule_type, holiday_date, holiday_date_end, holiday_name, 
   surcharge_per_person, surcharge_type, valid_year, currency, is_confirmed)
VALUES 
  ('[크루즈명]', '1N2D', '2026-04-30', '2026-05-01', '노동절 연휴', 
   1200000, 'per_person', 2026, 'VND', true),
  ('[크루즈명]', '1N2D', '2026-12-24', NULL, '크리스마스 이브', 
   1200000, 'per_person', 2026, 'VND', true),
  ('[크루즈명]', '1N2D', '2026-12-31', NULL, '연말연시', 
   1200000, 'per_person', 2026, 'VND', true);

-- Step 3: 검증
SELECT * FROM cruise_rate_card 
WHERE cruise_name = '[크루즈명]' AND valid_year = 2026;
```

## 7. 휴일 기간 신규 규칙 (2.0 업데이트)

### 변경사항

**기간형 휴일 저장 방식 개선**:
- **구 방식 ❌**: 각 날짜마다 행 분리 (2026/04/30과 2026/05/01 각 1행 = 2행)
- **신 방식 ✅**: 기간 사용 (holiday_date_end 활용 = 1행)

**컬럼명 수정**:
- `surcharge` → `surcharge_per_person` (명확한 컬럼명)

**추가 정보 입력**:
- `holiday_name`: 휴일명 (예: "노동절 연휴", "크리스마스 이브")
- `surcharge_type`: 'per_person' (고정)
- `is_confirmed`: true (확정 여부)

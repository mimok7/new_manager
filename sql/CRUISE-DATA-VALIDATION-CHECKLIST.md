# 크루즈 가격 데이터 표준 포맷 & 검증 체크리스트

## 📋 입수 데이터 포맷 표준화

### 1️⃣ 원본 데이터 형식 (카페/공식 포스팅)

일반적인 입수 형식:

```
[크루즈명]
예시: "얀센급 크루저 2026년 1박2일 객실요금"

[시즌 구분]
예시: 
- 2026년 02월 28일 까지
- 2026년 03월 01일 부터

[객실별 가격 표]
| 객실명 | 성인 | 아동 | 아동엑스 | 엑스트라 | 싱글차지 |
|--------|------|------|---------|--------|--------|

[휴일/성수기 추가요금]
예시:
- 1인당 1,500,000동 추가
- 기간: 2026/01/28~01/31
```

---

### 2️⃣ 데이터 정제 규칙

#### 객실명 정규화

| 원본 표기 | DB 저장 형식 | 예시 |
|---------|-----------|------|
| "오션스위트룸" | 그대로 입력 | 오션스위트룸 |
| "Ocean Suite Room" | 한글 우선 / 영문 secondary | 오션스위트룸 (Ocean Suite Room) |
| "더 오너스 스위트(2인)" | 정확히 입력 | 더 오너스 스위트 (2인) |
| "오너스-2인" | 표준형으로 변환 | 더 오너스 스위트 (2인) |

#### 가격 정규화

| 입력 형식 | DB 저장 | 주의사항 |
|---------|--------|--------|
| "5,250,000" | 5250000 | 쉼표 제거 |
| "5.250.000" (유럽식) | 5250000 | 점 제거, 쉼표 기준 |
| "5250,000동" | 5250000 | 단위/통화 제거 |
| "빈칸" (무료/불가) | NULL 또는 0 | 지침별 할당 값 |
| "불가" | NULL | 해당 옵션 미지원 |
| "성인요금 동일" | price_adult | price_child = price_adult |

#### 날짜 정규화

| 입력 | 저장 형식 | 예시 |
|------|---------|------|
| "2026/01/28" | YYYY-MM-DD | 2026-01-28 |
| "2026년 1월 28일" | YYYY-MM-DD | 2026-01-28 |
| "01/28~01/31" | holiday_date & holiday_date_end | 2026-01-28, 2026-01-31 |

## ✅ 각 크루즈별 입력 검증 포인트

### Diana (6 객실 × 2 시즌)

**필수 확인**:
- [ ] 객실 6개: 오션스위트룸, 트리플룸, 발코니, 베란다, 이그제큐티브, 에센스
- [ ] 시즌 2개: S1(1/1~4/30, 10/1~12/31), S2(5/1~9/30)
- [ ] 아동 기본값: S1 3,200,000 / S2 2,944,000 (8% 할인)
- [ ] 휴일: 3개 (크리스마스, 연말, 토요일)

**검증 표**:
```
시즌1 검증 = 객실 6개 × 가격 모두 입력
시즌2 검증 = 시즌1 가격 × 0.92 (8% 할인 계산)
```

---

### Lyra Granjer (11 객실 × 3 시즌)

**필수 확인**:
- [ ] 객실 11개:
  - General Suite: 4종
  - Family Suite: 5종
  - Luxury Suite: 2종
- [ ] 시즌 3개: S1(1/1~4/30), S2(5/1~9/30), S3(10/1~12/31)
- [ ] 특수: 유아요금 = 성인가 × 30% (자동 계산)
- [ ] Family Suite: price_child, price_extra_bed = NULL (패키지형)

**검증 표**:
```
General Suite 검증 = price_adult, price_child, price_infant, price_extra_bed, price_child_extra_bed 모두 입력
Family Suite 검증 = price_adult만 입력, 나머지 NULL
Luxury Suite 검증 = price_adult, price_infant, price_single 입력 가능
```

---

### Grand Fairyness (10 객실 × 2 시즌 + 정책변화)

**필수 확인**:
- [ ] 객실 10개:
  - General: 3종 (오션, 발코니, 트리플)
  - Upper: 2종 (베란다, 이그제큐티브)
  - Luxury: 2종 (에센스, 오셔니아)
  - Premium: 3종 (2인, 3인, 4인)
- [ ] 시즌 2개 + 정책: S1 (2/1~2/28) 정가 → S2 (3/1~12/31) 인하공지 (1/15)
- [ ] 특수: 럭셔리 객실 아동가는 인하 제외 (3,200,000 유지)

**검증 표**:
```
일반/상층 객실 검증 = S1 가격 VS S2 가격 (10%~15% 인하 범위)
럭셔리 객실 검증 = S1 아동가 == S2 아동가 (3,200,000 유지)
Premium 객실 검증 = price_child = price_adult (정원 기반 가격)
```

## 🔍 표 검증 절차 (데이터 수신 시)

### Phase 1: 구조 검증

```
입수된 표:
- 객실명 컬럼 있음? ✓
- 성인 가격 컬럼 있음? ✓
- 아동 가격 컬럼 있음? (또는 비어있음)
- 휴일 정보 섹션 있음? (있으면 분석)
- 기간 표기 있음? ✓
```

### Phase 2: 데이터 검증

```
각 객실:
- 모든 가격이 숫자인가? (또는 "불가", "동일")
- 가격 범위가 합리적인가? (0 < price < 50,000,000)
- 아동가 > 성인가 또는 이상한 값? → 확인 필요

전체:
- 객실 개수가 기대값과 일치? (지침 기준)
- 시즌 개수가 기대값과 일치?
- 휴일 추가요금이 기대값과 일치?
```

### Phase 3: 규칙 검증

```
지침 규칙 적용:
- 아동 기본값이 올바르게 할당되었나? (시즌별)
- 특수 객실 (불가, 동일) 처리가 맞나?
- 패키지형 객실은 price_child = price_adult?
- 럭스 객실과 일반 객실의 정책 차이가 반영됨?
```

## ⚠️ 일반적인 입력 오류 & 해결책

| 오류 | 원인 | 해결 |
|------|------|------|
| 아동가 > 성인가 | 입력 실수 또는 특수 정책 | 지침 확인, 감수자 연락 |
| 시즌 개수 불일치 | 표 누락 또는 중복 | 포스팅 원본 재확인 |
| 객실명 오타 | 손 입력 또는 OCR 오류 | 원본 스크린샷 검증 |
| 기간 표기 불명확 | "~까지", "부터" 혼동 | 포스팅 문맥 재확인 |
| 휴일 추가요금 누락 | 별도 섹션 미발견 | 전체 포스팅 재검색 |

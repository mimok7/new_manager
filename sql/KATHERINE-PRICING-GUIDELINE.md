## DB 저장 규칙

### cruise_rate_card 테이블 구조

```
- cruise_name: '캐서린 크루즈'
- schedule_type: '1N2D'
- room_type: 객실명 (한글)
- room_type_en: 객실명 (영문, 옵션)
- price_adult: 성인 요금
- price_child: 아동 요금 (불가시 NULL)
- price_infant: NULL (미사용)
- price_extra_bed: 엑스트라베드 요금 (없으면 NULL)
- price_child_extra_bed: NULL (미사용)
- price_single: 싱글차지 (불가시 NULL)
- extra_bed_available: true/false
- valid_year: 2026
- currency: 'VND'
- is_active: true
- valid_from: YYYY-MM-DD
- valid_to: YYYY-MM-DD
- notes: 객실 특성 및 제약사항
```

### cruise_holiday_surcharge 테이블 구조 (휴일 추가요금)

```
- cruise_name: '캐서린 크루즈'
- schedule_type: '1N2D'
- holiday_date: YYYY-MM-DD (시작일)
- holiday_date_end: YYYY-MM-DD (종료일, 기간형만 입력하고 단일일은 NULL)
- holiday_name: 휴일명 (노동절 연휴, 크리스마스 이브 등)
- surcharge_per_person: 인당 추가요금
- surcharge_type: 'per_person' (고정)
- valid_year: 2026
- currency: 'VND'
- is_confirmed: true (확정)
```

### 데이터 저장 패턴

**객실요금**:
1. **총 객실**: 7개 = 7행
2. **각 행 구성**: 
   - room_type (객실명) = 유니크 키
   - 모든 카테고리 요금을 컬럼에 저장
   - notes에 특이사항 기록

**휴일 추가요금**:
1. **총 기간**: 3개 = 3행 (기간형 1 + 단일형 2)
2. **기간형 표시** (예: 노동절 연휴):
   - `holiday_date = 2026-04-30`
   - `holiday_date_end = 2026-05-01`
3. **단일형 표시** (예: 크리스마스):
   - `holiday_date = 2026-12-24`
   - `holiday_date_end = NULL`

## 특이사항

1. **프리미어 스위트 트리플**: 아동 불가, 엑스트라베드 불가, 싱글차지 불가
2. **엑스트라베드 계산**: 추가요금 (성인 2인 + 엑스트라 = 총 비용)
3. **빌라 프레지던트**: 아동 가격이 매우 저가 (성인 대비 22%)
4. **휴일 추가요금**: 
   - 노동절 연휴 (2026/04/30 ~ 2026/05/01): 기간형 1행
   - 크리스마스 이브 (2026/12/24): 단일형 1행
   - 연말연시 (2026/12/31): 단일형 1행
   - 총 3행 저장

# 크루즈 지침 업데이트 프로세스

## 🎯 개요

SQL 실행 후 **각 크루즈별 가격 지침 문서**를 최신 상태로 업데이트하는 절차입니다.

---

## 📋 지침 문서 구조

각 크루즈마다 다음과 같은 구조를 유지합니다:

```
{크루즈명}-PRICING-GUIDELINE.md
├── 📝 개요 (크루즈명, 여행형태, 통화, 연도)
├── 🎯 데이터 구조 (시즌 정의, 객실 목록, 가격 규칙)
├── 📊 시즌별 가격표 (실제 DB에 들어간 데이터)
├── 🏨 객실 상세정보 (특수성, 옵션 가능/불가)
├── 🎄 휴일 추가요금
├── 📝 변경 이력 (언제, 누가, 뭘 했는지)
└── ✅ 검증 체크리스트
```

---

## 🔄 업데이트 절차 (Step-by-Step)

### Step 1: 지침 파일 열기

**위치**: `sql/{크루즈명}-PRICING-GUIDELINE.md`

**예**:
- `sql/GRAND-FAIRYNESS-PRICING-GUIDELINE.md`
- `sql/DIANA-PRICING-GUIDELINE.md`

---

### Step 2: "변경 이력" 섹션 업데이트

**현재 이력 예시**:
```markdown
## 📝 변경 이력

| 날짜 | 시즌 | 변경 내용 | 상태 |
|------|------|---------|------|
| 2026-02-21 | S1 (2026/02) | 초기 데이터 입력 | ✅ 완료 |
```

**새 항목 추가** (맨 위에 추가):
```markdown
| 2026-02-21 | S1, S2 | 시즌2 추가 및 가격 인하 반영 | ✅ 완료 |
| 2026-02-21 | S1 (2026/02) | 초기 데이터 입력 | ✅ 완료 |
```

---

### Step 3: "시즌별 가격표" 업데이트 (실제 DB 데이터 반영)

**변경 전**:
```markdown
### 📊 시즌1: 2026/02/01 - 02/28 (정가)

| 객실명 | 성인 | 아동 | 엑스트라 | 싱글차지 |
|--------|------|------|--------|--------|
| 오션스위트 | 5,250,000 | 3,200,000 | 4,800,000 | 8,600,000 |
```

**변경 후** (DB에서 조회 후 복사):
```markdown
### 📊 시즌1: 2026/02/01 - 02/28 (정가)

| 객실명 | 성인 | 아동 | 아동엑스 | 엑스트라 | 싱글차지 |
|--------|------|------|---------|--------|--------|
| 오션스위트룸 | 5,250,000 | 3,200,000 | 4,250,000 | 4,800,000 | 8,600,000 |
| 오션스위트 트리플룸 | 5,300,000 | 불가 | 불가 | 불가 | 불가 |
| 오션발코니 스위트 | 5,250,000 | 3,200,000 | 4,250,000 | 4,800,000 | 8,600,000 |
```

**데이터 소스**: Supabase 검증 쿼리 결과
```sql
SELECT 
  room_type,
  price_adult,
  price_child,
  price_child_extra_bed,
  price_extra_bed,
  price_single
FROM cruise_rate_card
WHERE cruise_name = '그랜드 파이어니스 크루즈'
  AND schedule_type = '1N2D-S1'
ORDER BY room_type;
```

---

### Step 4: "휴일 추가요금" 섹션 업데이트

**변경 전**:
```markdown
## 🎄 휴일/성수기 추가요금

### 명절 연휴
- 기간: 2026-01-28 ~ 2026-01-31
- 1인당 추가요금: 1,500,000동
```

**변경 후** (DB에서 조회):
```markdown
## 🎄 휴일/성수기 추가요금

| 기간 | 휴일명 | 시작 | 종료 | 1인당추가 |
|------|--------|------|------|---------|
| 명절 | 명절 연휴 | 01-28 | 01-31 | 1,500,000 |
| 구정 | 구정 연휴 | 02-15 | 02-21 | 1,600,000 |
| 노동절 | 노동절 연휴 | 04-30 | 05-01 | 1,500,000 |
| 추석 | 추석 연휴 | 08-30 | 09-01 | 1,250,000 |
| 크리스마스 | 크리스마스 이브 | 12-24 | - | 1,500,000 |
| 연말 | 연말연시 | 12-31 | - | 1,500,000 |
```

**데이터 소스**:
```sql
SELECT
  holiday_date,
  holiday_date_end,
  holiday_name,
  surcharge_per_person
FROM cruise_holiday_surcharge
WHERE cruise_name = '그랜드 파이어니스 크루즈'
ORDER BY holiday_date;
```

---

### Step 5: "객실 상세정보" 섹션 검토

새로운 객실이 추가되었거나 특수성이 변경되면 업데이트:

**예: Grand Fairyness의 특수성**

```markdown
## 🏨 객실 상세 정보

### 일반 객실 (General)
- **오션스위트룸**, **오션발코니 스위트**: 
  - 아동 및 엑스트라베드 지원
  - 아동 엑스트라베드 가격: 성인 엑스트라베드의 88.5%

### 럭셔리 객실 (Luxury)
- **더 에센스 스위트**, **더 오셔니아 스위트**:
  - 아동 불가 또는 제한적 지원
  - 시즌 간 아동가 유지 (3,200,000동 고정)
  - 인하 정책 제외 대상

### 프리미엄 객실 (Premium)
- **더 오너스 스위트 (2인/3인/4인)**:
  - 아동가 = 성인가 (정원 기반)
  - 엑스트라베드 미지원
```

---

### Step 6: "DB 매핑" 섹션 업데이트 (내부용)

```markdown
## 🗄️ DB 테이블 매핑

### cruise_rate_card
- **cruise_name**: '그랜드 파이어니스 크루즈'
- **schedule_type**: '1N2D' (또는 '1N2D-S1', '1N2D-S2')
- **room_type**: 객실명 (예: '오션스위트룸')
- **room_type_en**: 영문명 (예: 'Ocean Suite Room')
- **valid_from** ~ **valid_to**: 시즌 기간
- **금액 컬럼**:
  - price_adult: 기본 성인가
  - price_child: 아동가 (또는 NULL)
  - price_extra_bed: 성인 엑스트라베드
  - price_child_extra_bed: 아동 엑스트라베드 (또는 NULL)

### 최근 업데이트
```
실행 일시: 2026-02-21 14:30:00 KST
영향 행 수: 20 (객실 가격)
휴일 행 수: 6 (추가요금)
```

---

### Step 7: "변경 사항 요약" 추가 (선택사항)

**새로운 섹션 추가**:

```markdown
## 📌 주요 변경 사항 (2026-02-21)

### 가격 정책 변화
- 시즌1 (02/01-02/28): 정가 유지
- 시즌2 (03/01-12/31): **일부 할인 적용**
  - 일반/상층 객실: 30,000~100,000동 인하
  - 럭셔리 객실: 가격 유지

### 신규 추가
- 없음

### 폐지 항목
- 없음

### 영향 범위
- 총 객실: 10개 (변동 없음)
- 총 휴일: 6개 (변동 없음)
```

---

### Step 8: 최종 검증 체크리스트

**지침 업데이트 완료 후 확인**:

```markdown
## ✅ 지침 업데이트 검증

- [ ] 시즌별 가격표 최신 데이터 반영
- [ ] 객실명 오타 없음
- [ ] 휴일 추가요금 테이블 완성
- [ ] 특수 케이스 설명 추가 (있으면)
- [ ] 변경 이력 날짜 기록
- [ ] DB 매핑 정보 검토
- [ ] 문법/포맷 검토 (Markdown)
- [ ] 리뷰어 확인 완료
```

## 📝 각 크루즈별 지침 업데이트 예시

### 예 1: Grand Fairyness (시즌 추가)

**상황**: 시즌2 데이터 추가, 일부 가격 인하

**업데이트 항목**:
1. ✅ 변경 이력: "2026-02-21 | S1, S2 | 시즌2 추가..." 항목 추가
2. ✅ 가격표: 시즌1, 시즌2 두 테이블 모두 표시
3. ✅ 특수성: 럭셔리 객실 아동가 유지 설명 추가
4. ✅ 휴일: 6개 추가요금 테이블 표시

---

### 예 2: Diana (8% 할인)

**상황**: 기존 1 시즌에서 2 시즌 추가, S2는 S1의 92%

**업데이트 항목**:
1. ✅ 변경 이력: "시즌2 추가, 8% 할인" 기록
2. ✅ 가격표: 시즌1(정가), 시즌2(92% 계산) 명확히
3. ✅ 주요 변경: "계절할인 정책 적용" 섹션 추가

---

### 예 3: 새 크루즈 추가 (Paradise Legacy)

**상황**: 완전히 새로운 크루즈 추가

**생성 항목**:
1. 📄 새 지침 파일 생성: `PARADISE-LEGACY-PRICING-GUIDELINE.md`
2. ✅ 기본 정보: 크루즈명, 형태, 통화 작성
3. ✅ 시즌 정의: 해당 연도 시즌 명시
4. ✅ 객실 목록: 모든 객실명 나열
5. ✅ 가격표: 시즌별 완성 테이블
6. ✅ 휴일: 해당 추가요금
7. ✅ 변경 이력: "2026-02-21 | 초기 생성" 기록

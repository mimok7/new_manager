---

## 1. 프로젝트 개요

**크루즈명**: 그랜드 파이어니스 크루즈 (Grand Fairyness Cruise)  
**상품타입**: 2박 3일 (2N3D)  
**통화**: VND 베트남동  
**연도**: 2026년  
**시즌 구분**: 없음 (연중 동일 요금)  
**결제 방법**: 2가지 (신용카드 vs VND 송금)

---

## 2. 결제 방법별 정의

### 2.1 신용카드 결제 (Credit Card Payment)

- **표준 결제 방식**
- **가격**: 높음 (기본 요금)
- **예**: 오션스위트룸 신용카드 = 9,850,000동
- **DB 표현**: room_type에 "(신용카드)" 표기

### 2.2 VND 송금 결제 (VND Transfer Payment)

- **베트남동 직송금 방식** (트레블월릿 등 APP 사용)
- **가격**: 신용카드 대비 2~5% 저렴
- **예**: 오션스위트룸 VND송금 = 9,600,000동 (약 2.5% 저렴)
- **DB 표현**: room_type에 "(VND송금)" 표기

## 4. 데이터 구조

### 4.1 cruise_rate_card 테이블

**목적**: 기본 객실 요금 관리 (결제 방법 구분)

**season_name 컬럼의 역할**:
- **신용카드**: 신용카드 결제 방식의 객실 (가격 높음)
- **VND 송금**: VND 송금 결제 방식의 객실 (가격 저렴)
- 결제 방법별 데이터를 구분하는 **주요 식별자** 역할
- 검증 쿼리에서 `WHERE season_name = '신용카드'` 또는 `WHERE season_name = 'VND 송금'`으로 필터링

| 컬럼 | 값 | 비고 |
|------|-----|------|
| cruise_name | 그랜드 파이어니스 크루즈 | 고정값 |
| schedule_type | 2N3D | 2박 3일 구분자 |
| room_type | 오션스위트룸 (신용카드), 오션스위트룸 (VND송금) | 결제방법 구분 포함 |

## 5. 주요 특징 및 정책

### 5.1 결제 방법별 가격 정책

**신용카드 vs VND 송금 할인율**

| 객실명 | 신용카드 | VND송금 | 할인율 |
|--------|---------|---------|--------|
| 오션스위트룸 | 9,850,000 | 9,600,000 | 2.5% |
| 베란다 스위트 | 11,750,000 | 11,400,000 | 3.0% |
| 이그제큐티브 스위트 | 12,750,000 | 12,400,000 | 2.7% |
| 더 에센스 스위트 | 17,500,000 | 17,000,000 | 2.9% |
| 더 오셔니아 스위트 | 29,000,000 | 28,300,000 | 2.4% |
| 더 오너스 2인 | 43,500,000 | 42,300,000 | 2.8% |
| 더 오너스 3인 | 32,500,000 | 31,700,000 | 2.5% |
| 더 오너스 4인 | 26,900,000 | 26,200,000 | 2.6% |

### 5.2 객실별 특수 조건

| 객실명 | 특징 | 엑스트라베드 | 아동 정책 |
|--------|------|-----------|---------|
| 오션스위트 트리플 | 기본 침대 3개 | 불가 | 동일 |
| 더 에센스 스위트 | 럭셔리 객실 | 불가 | 동일 |
| 더 오셔니아 스위트 | 최상급 객실 | 불가 | 동일 |
| 더 오너스 (2/3/4인) | 프리미어 객실 | 불가 | **성인과 동일** |

**더 오너스 스위트 특수 정책**:
- price_child = price_adult (1:1 비율)
- 3인 객실: 3인 기준 요금으로 책정
- 4인 객실: 4인 기준 요금으로 책정
- 추가 침대 없음 (정원제)

### 5.3 아동 요금 정책

**대부분 객실**:
- 신용카드: 5,900,000동 (약 60% of adult)
- VND송금: 5,750,000동 (약 60% of adult)

**더 오너스 스위트**:
- 모든 숙박자가 동일 요금 (아동/성인 구분 없음)
- 예: 2인 객실 1박 → 2 × 43,500,000동 = 87,000,000동

**중요**: 결제 방법은 DB에서 구분되며, 삭제 시 schedule_type으로 필터링되므로 안전합니다.

## 7. 검증 쿼리

### 7.1 신용카드 결제 요금 확인 (season_name='신용카드' 기준)

```sql
SELECT 
  season_name as "결제방법",
  room_type as "객실명",
  price_adult as "성인",
  price_child as "아동",
  price_extra_bed as "엑스트라베드",
  CASE WHEN extra_bed_available THEN 'O' ELSE 'X' END as "추가가능"
FROM cruise_rate_card
WHERE cruise_name = '그랜드 파이어니스 크루즈'
  AND schedule_type = '2N3D'
  AND season_name = '신용카드'
ORDER BY price_adult DESC;
```

**예상 결과**: 10행 (season_name='신용카드'인 객실)

### 7.2 VND 송금 결제 요금 확인 (season_name='VND 송금' 기준)

```sql
SELECT 
  season_name as "결제방법",
  room_type as "객실명",
  price_adult as "성인",
  price_child as "아동",
  price_extra_bed as "엑스트라베드",
  CASE WHEN extra_bed_available THEN 'O' ELSE 'X' END as "추가가능"
FROM cruise_rate_card
WHERE cruise_name = '그랜드 파이어니스 크루즈'
  AND schedule_type = '2N3D'
  AND season_name = 'VND 송금'
ORDER BY price_adult DESC;
```

**예상 결과**: 10행 (season_name='VND 송금'인 객실)

### 7.3 휴일 추가요금 확인

```sql
SELECT
  holiday_date as "시작일",
  holiday_date_end as "종료일",
  holiday_name as "휴일명",
  surcharge_per_person as "성인추가요금"
FROM cruise_holiday_surcharge
WHERE cruise_name = '그랜드 파이어니스 크루즈'
  AND schedule_type = '2N3D'
ORDER BY holiday_date;
```

**예상 결과**: 6행

### 7.4 데이터 개수 확인

```sql
SELECT 
  (SELECT COUNT(*) FROM cruise_rate_card 
   WHERE cruise_name = '그랜드 파이어니스 크루즈' AND schedule_type = '2N3D') as "객실가격행 (20행 예상)",
  (SELECT COUNT(*) FROM cruise_holiday_surcharge 
   WHERE cruise_name = '그랜드 파이어니스 크루즈' AND schedule_type = '2N3D') as "휴일추가행 (6행 예상)";
```

**예상 결과**: 객실 20행, 휴일 6행

### 7.5 결제 방법별 season_name 확인

```sql
SELECT 
  season_name as "결제방법",
  COUNT(*) as "객실수"
FROM cruise_rate_card
WHERE cruise_name = '그랜드 파이어니스 크루즈'
  AND schedule_type = '2N3D'
GROUP BY season_name
ORDER BY season_name;
```

**예상 결과**:
- 신용카드: 10행
- VND 송금: 10행

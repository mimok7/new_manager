# 그랜드 파이어니스 크루즈 데이터 입력 지침

## 📋 개요

**크루즈명**: 그랜드 파이어니스 크루즈  
**여행 형태**: 1박 2일 (1N2D)  
**통화**: VND (베트남동)  
**유효년도**: 2026년  

---

## 🎯 데이터 구조 및 패턴

### 입력 형식 요구사항

그랜드 파이어니스 크루즈는 **카페 포스팅 데이터**에서 다음 정보를 추출합니다:

#### 1. 시즌 분리
- **시즌 1**: 객실요금 표의 "~년 ~월 ~일 까지" 기한
- **시즌 2**: 객실요금 표의 "~년 ~월 ~일 부터" 시작

#### 2. 객실별 가격 정보 테이블 (월별 2개 테이블)

| 컬럼 | 의미 | 입력 규칙 |
|------|------|---------|
| 객실명 | 크루즈 객실 이름 | 정확히 입력 |
| 성인 1인당 | 성인 기본 가격 | 숫자만 입력 |
| 아동 1인당 | 아동 가격 | 아래 규칙 따름 |
| 아동 엑스트라 베드 | 아동이 엑스트라베드 사용 시 가격 | 아래 규칙 따름 |
| 엑스트라베드 | 성인용 엑스트라베드 가격 | 아래 규칙 따름 |
| 1인 싱글차지 | 1인 싱글 객실 요금 | 아래 규칙 따름 |

### 아동 가격 해석 규칙 (CORRECTED)

**"아동 1인당" 컬럼 해석**:

| 원본 표기 | DB 처리 방식 | 설명 |
|---------|-----------|------|
| **숫자** | 그대로 입력 | 예: 3,200,000 → 3200000 |
| **빈칸 (비어있음)** | 기본값 입력 | S1: 3,200,000 / S2: 3,150,000 |
| **"성인요금 동일"** | price_adult 값과 동일 | 오너스 시리즈 (기준 인원별 가격) |

**✅ 변경사항**: 
- 모든 객실이 아동가를 가짐 (기본값 또는 명시값)
- 트리플, 에센스, 오셔니아도 아동가 입력됨

**"아동 엑스트라 베드" 컬럼 해석**:

| 원본 표기 | DB 처리 방식 | 설명 |
|---------|-----------|------|
| **숫자** | 그대로 입력 | 예: 4,250,000 → 4250000 |
| **빈칸** | NULL | 아동엑스트라 미지원 |
| **"-" (하이픈)** | NULL | 명시적 불가 표기 |

**"엑스트라베드" 컬럼 해석**:

| 원본 표기 | DB 처리 방식 | 설명 |
|---------|-----------|------|
| **숫자** | 그대로 입력 | 일반적인 경우 |
| **빈칸** | NULL | 엑스트라베드 옵션 없음 |
| **"-" (하이픈)** | NULL | 명시적 불가 표기 |

**"1인 싱글차지" 컬럼 해석**:

| 원본 표기 | DB 처리 방식 | 설명 |
|---------|-----------|------|
| **숫자** | 그대로 입력 | 싱글 객실 요금 |
| **빈칸** | NULL | 싱글차지 불가 |
| **"-" (하이픈)** | NULL | 명시적 불가 표기 |

## 📊 데이터 매핑 예시

### 시즌 1 예시 (그랜드 파이어니스)

원본 표 데이터:
```
객실명: 오션스위트 트리플룸
성인 1인당: 5,300,000
아동 1인당: [빈칸]
아동 엑스트라 베드: -
엑스트라베드: -
1인 싱글차지: -
```

DB 입력:
```
price_adult: 5300000
price_child: 3200000          ← 빈칸이므로 기본값 (시즌1)
price_infant: NULL
price_extra_bed: NULL         ← 하이픈 = NULL
price_child_extra_bed: NULL   ← 하이픈 = NULL
price_single: NULL            ← 하이픈 = NULL
```

### 시즌 2 예시 (그랜드 파이어니스)

원본 표 데이터:
```
객실명: 오션발코니 스위트
성인 1인당: 5,200,000
아동 1인당: 4,200,000         ← 시즌2에서는 수치 명시
아동 엑스트라 베드: 4,200,000
엑스트라베드: 4,700,000
1인 싱글차지: 8,600,000
```

DB 입력:
```
price_adult: 5200000
price_child: 4200000          ← 명시된 값 사용 (아동가)
price_infant: NULL
price_extra_bed: 4700000
price_child_extra_bed: 4200,000  ← 아동엑스트라 베드 가격
price_single: 8600000
```

### 특수 케이스

#### 1. 아동 불가 객실 (트리플룸, 에센스, 오셔니아)

원본 (시즌1):
```
객실명: 오션스위트 트리플룸
아동 1인당: 불가
아동 엑스트라 베드: [빈칸]
엑스트라베드: [빈칸]
1인 싱글차지: [빈칸]
```

DB 입력:
```
price_child: NULL
price_child_extra_bed: NULL
price_extra_bed: NULL
price_single: NULL
extra_bed_available: false
```

#### 2. 성인요금 동일 객실 (오너스 시리즈)

원본 (시즌1):
```
객실명: 더 오너스 스위트 (2인)
성인 1인당: 22,900,000
아동 1인당: 성인요금 동일
아동 엑스트라 베드: [빈칸]
엑스트라베드: 불가
1인 싱글차지: [빈칸]
```

DB 입력:
```
price_child: 22900000        ← 성인가와 동일
price_child_extra_bed: NULL
price_extra_bed: NULL
price_single: NULL
```

## 🗄️ DB 테이블 컬럼 매핑

### cruise_rate_card (객실 요금)

| DB 컬럼 | 값 | 출처 |
|--------|-----|------|
| `cruise_name` | '그랜드 파이어니스 크루즈' | 고정 |
| `schedule_type` | '1N2D-S1' 또는 '1N2D-S2' | 시즌 분류 |
| `room_type` | 객실명 | 객실명 칼럼 |
| `room_type_en` | 영문 객실명 | 영문 변환 |
| `price_adult` | 성인 1인당 | 그대로 입력 |
| `price_child` | 아동 가격 해석 | 위의 규칙 따름 |
| `price_infant` | NULL | 미사용 (유아 정책별도) |
| `price_extra_bed` | 엑스트라베드 가격 | 위의 규칙 따름 |
| `price_child_extra_bed` | 아동 엑스트라 베드 | 위의 규칙 따름 |
| `price_single` | 1인 싱글차지 | 위의 규칙 따름 |
| `extra_bed_available` | true/false | 엑스트라베드 가능 여부 |
| `valid_year` | 2026 | 연도 |
| `valid_from` | 시즌 시작일 | 포스팅 기한 |
| `valid_to` | 시즌 종료일 | 포스팅 기한 |
| `currency` | 'VND' | 고정 |
| `is_active` | true | 고정 |

### cruise_holiday_surcharge (휴일 추가요금)

원본 포스팅의 "추가" 섹션 해석:

| 원본 표기 | DB 구조 | 설명 |
|---------|--------|------|
| 2026/01/28 부터 01/31 | holiday_date: 2026-01-28, holiday_date_end: 2026-01-31 | 기간형 |
| 2026/12/24 | holiday_date: 2026-12-24, holiday_date_end: NULL | 단일형 |

| DB 컬럼 | 값 | 규칙 |
|--------|-----|------|
| `cruise_name` | '그랜드 파이어니스 크루즈' | 고정 |
| `schedule_type` | '1N2D' | 고정 (모든 방문에 적용) |
| `holiday_date` | 기간 또는 단일 날짜 | 포스팅 데이터 |
| `holiday_date_end` | 기간 종료날 또는 NULL | 포스팅 데이터 |
| `holiday_name` | 휴일 명칭 | 포스팅에서 추출 |
| `surcharge_per_person` | 추가 금액 | "1인당 ~동 추가" |
| `surcharge_type` | 'per_person' | 고정 |
| `valid_year` | 2026 | 고정 |
| `currency` | 'VND' | 고정 |
| `is_confirmed` | true | 공식 공지이므로 true |

## ✅ 최종 결과물 검증

SQL 실행 후 다음 결과를 생성해야 함:

### 1. 시즌별 객실 가격표 (GROUP BY schedule_type)

```
시즌1: 10개 객실 × 1행 = 10행
시즌2: 10개 객실 × 1행 = 10행
합계: 20행
```

### 2. 가격 검증 항목

- [ ] 모든 객실의 성인가 입력됨
- [ ] 아동 불가 객실은 price_child = NULL
- [ ] 빈칸 객실은 기본값 입력 (S1: 3,200,000 / S2: 3,150,000)
- [ ] 성인요금 동일 객실은 price_child = price_adult
- [ ] 엑스트라 불가 객실은 price_extra_bed = NULL
- [ ] 휴일 추가요금 6개 모두 입력됨
- [ ] 휴일 기간형은 holiday_date_end 값 있음
- [ ] 휴일 단일형은 holiday_date_end = NULL
